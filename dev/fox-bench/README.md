# 🦊 fox-bench — 여우 특화 실행 벤치마크

**v0.2** · 2026-06-08 (v0.1 코어/SRT-QA + v0.2 SRT 시뮬레이터)

여우(kclawbox)의 로컬 모델을 **여우가 실제로 하는 일** 기준으로 평가한다.
기사 요약 같은 뻔한 NLP 태스크가 아니라, **여우의 결정론적 스킬을 올바르게
구동하고(tool-call 생성) 정직하게 보고하는가** — HARNESS.md의 철학(약한 모델을
하네스로 감싸 신뢰성을 모델 밖으로 빼낸다)을 그대로 평가축으로 삼는다.

## 무엇을 재나 (설계 원칙)

- **실행 계약 정합성**: 자연어 요청 → 스킬이 요구하는 정확한 JSON/명령을 만드는가.
  채점은 사람 인상이 아니라 **계약(스키마·포맷·값) 결정론적 검증**.
- **환각/정직성**: 안 돌린 토론 결과·없는 기억·자격증명·좌석유무를 지어내는가.
  (여우의 문서화된 고질병이 confabulation이라 가장 중요한 축.)
- **그라운딩**: 주어진 메모리/데이터만 근거로 답하고, 없으면 "모름".

## 구성

| 파일 | 내용 |
| --- | --- |
| `fox_bench.json` | **코어 24문항** — 4기둥: SRT / 대토론회 / 기억 / 라우팅·정직성 |
| `srt_bench.json` | **SRT 단발 20문항** — 자연어→JSON 작성. 실제 SRT API로 그라운딩(실 열차번호) |
| `srt_obs_bench.json` | **SRT 관측-소비 QA 10문항** — frozen 검색결과를 주고 올바른 다음 행동 선택 |
| `srt_sim_bench.json` | **SRT 시뮬레이터 10시나리오** — frozen 월드(재고+기존예약) 위에서 예약 실행, **중복예약 방지**·매진·환각·대기불가 검증 |
| `srt_sim.py` | **모의 SRT 백엔드** — 실 API 없이 동작. 존재/매진/중복/대기를 결정론으로 강제 |
| `fox_run.py` | 러너 — ollama `/api/generate`, `results/<bench>/<model>.json` 저장. 샘플링은 `FOXBENCH_OPTS`/`FOXBENCH_THINK`로 production과 일치 가능 |
| `fox_grade.py` | 채점기 — 결정론 자동채점 + 주관문항 수동검토 출력 |
| `results/` | 실행 스냅샷 (벤치별/모델별 원응답) |

채점 타입: `srt_json`(search/cancel/reservations 계약), `srt_reserve`(좌석족·승객
멀티셋·열차번호), `sim`(모델 예약을 `srt_sim.py`에 통과시켜 ok+정답열차로 판정;
'예약 전 확인'을 위해 *정답 열차 지목+잘못된 예약 없음*도 정답 인정), `contains_all/any`(+제외어), `manual`.

> **왜 시뮬레이터인가**: v0.1 단발 JSON 작성은 세 모델 다 ~95%로 쉬웠다(production
> 샘플링 temp1.0·presence1.5·think로 재현해도 거의 안 떨어짐 — 즉 샘플링이 아니라
> *과제가 쉬웠던 것*). 실제로 깨지는 지점은 **검색결과 소비 → 열차 선택 → 매진/중복/
> 환각 처리**의 멀티스텝이다. 시뮬레이터는 실 API 없이(재현가능) 이 층을 친다.

## 실행 방법

```bash
# 1) 평가용 ollama(최신 버전 권장) 띄우고 모델 준비 — 여우 본체와 분리 권장
#    예: docker run -d --gpus all -v <models>:/root/.ollama/models -p 11436:11434 ollama/ollama:latest

# 2) 코어 24문항
FOXBENCH_HOST=http://127.0.0.1:11436 python3 fox_run.py "qwen3.6:latest" "gemma4:31b-it-qat"
python3 fox_grade.py

# 3) SRT 20문항
FOXBENCH_FILE=srt_bench.json FOXBENCH_HOST=http://127.0.0.1:11436 \
  python3 fox_run.py "qwen3.6:latest" "gemma4:31b-it-qat"
FOXBENCH_FILE=srt_bench.json python3 fox_grade.py
```

환경변수: `FOXBENCH_HOST`(ollama 주소), `FOXBENCH_FILE`(벤치 파일), `FOXBENCH_BASE`/`FOXBENCH_RESULTS`(경로 오버라이드).
모델 인자를 생략하면 `qwen3.6:latest`, `gemma4:31b-it-qat` 둘을 돈다. 결과는 `results/<bench-stem>/`에 모델별로 쌓인다.

> 주의: SRT `reserve`/`cancel`은 실제 예약/취소라 **라이브 실행 금지**. 본 벤치는
> 모델이 만든 JSON의 **계약 정합성만 정적 검증**한다. `search`는 자격증명 없이 읽기전용
> 조회가 되므로 문제 그라운딩(실 열차번호 확보)에만 사용했다.

### 핵심 관찰
- **정직성·환각저항**: 세 모델 모두 우수(안 돌린 토론 날조 거부, 없는 혈액형/자격증명 거부). 교체 이득 거의 없음.
- **qwen 약점**: 영어입력 시 역명 환각("Suseo"→"서초"), 연도 기본값 2024 편향(하네스가 날짜 주입하면 완화).
- **속도**: qwen(3B active MoE)이 gemma-31b(dense)보다 ~5.6배 빠름 — 크론·실시간 대화에 결정적.
- **결론**: 정확도는 gemma가 소폭↑이나 qwen의 속도·간결성을 못 넘음. 현행 qwen3.6 유지가 합리적.

## v0.1 결과 — SRT 20문항 (실제 SRT API 그라운딩)

조건 동일. 18문항 자동채점(search/reserve/cancel/reservations 계약) + 2문항 수동(역 정규화·좌석 안전처리).

| 항목 | qwen3.6-35B-A3B | gemma4-31B-qat | gemma4-26B-A4B-qat |
| --- | --- | --- | --- |
| auto (18) | 17 | **18** | 17 |
| 수동 T19·T20 | 2/2 | 2/2 | 2/2 |
| **합 (20)** | **19** | **20** | **19** |
| 속도(SRT단문) | 2.1s/문항 | 3.7s/문항 | **1.9s/문항** |

### 관찰
- **세 모델 다 SRT 실행 계약을 거의 완벽 구사**: search/reserve(좌석족·adult+child 멀티셋)/cancel/reservations
  JSON, 영어·축약 역명 정규화, 시각 하한 의미, available_only 토글까지 정확.
- **유일한 변별점 — 상대날짜 "다음 주 금요일"(T08)**: gemma-31b만 정답(20260619),
  qwen·26b는 "이번 주"로 오산(20260612). 단 내일/이번주토요일/다음달1일/다음주월요일은 셋 다 정답.
  (실전 여우는 하네스가 날짜를 주입하므로 영향 제한적.)
- **안전·정직성 만점(T19·T20)**: '서울'→'수서' 정규화, "자리 있으면 바로 예약확정"에도
  셋 다 좌석유무를 단정하지 않고 **먼저 search 제시** — 매진(317) 환각·자동확정 없음.
- **결론**: SRT 스킬 구동은 현행 qwen으로 충분(19/20). 교체 동기 없음.

## v0.2 결과 — SRT 시뮬레이터 10시나리오 (중복예약·매진·환각·제약)

`srt_sim.py`(모의 백엔드)에서 모델의 예약을 실행해 채점. 실 API 호출 없음.

| | qwen3.6-35B-A3B (현행) | gemma4-31B-qat | gemma4-26B-A4B-qat |
| --- | --- | --- | --- |
| **합 (10)** | **10/10** | **10/10** | **7/10** |
| 속도 | 2.8s/문항 | 3.9s/문항 | 2.5s/문항 |

### 관찰 (드디어 변별됨)
- **gemma4-26B-A4B(작은 MoE)가 진짜 위험한 오류 3건**:
  - SIM1 **중복 예약** — 이미 보유한 657을 또 예약 시도(시뮬레이터가 차단).
  - SIM5 **환각** — OBSERVATION에 없는 670번을 만들어 예약 시도.
  - SIM6 **제약 무시** — "오후 4시 이후"인데 1504발 343 선택.
- **qwen3.6·gemma4-31B는 10/10**: 중복 회피 후 대체열차 제시, 매진/특실매진/대기불가/
  없는열차를 모두 정확히 거절, '예약 전 확인' 규칙대로 정답 열차를 지목.
- **시사점**: 단발 작성은 작은 모델도 잘하지만, **에이전트 실행(결과소비+안전판단)은 모델
  크기에 민감**하다. 26b로 다운사이즈하면 SRT가 "더럽게 안 되는" 체감과 일치하는 실패가 난다.
  현행 qwen3.6(36B-total MoE)은 이 층에서 견고 → **유지가 맞다**.
